ARC-AGI2 is designed to resist brute-force scaling - yet Gemini 3 Deep Think now matches average human performance. METR's time horizon metric doubled from 4 minutes to 14.5 hours in under two years, doubling every ~123 days. At that rate, week-long autonomous tasks become plausible by late 2026. The models are not the bottleneck anymore.

Remaining Fundamental Challenges

  • Long-term Context Management
  • Online Learning
  • Hallucinations

Everything else is tractable engineering - and it more than compensates.

Context: models have finite context windows and no true memory across sessions - engineering workarounds (compaction, memory files, RAG) help but there's still a discontinuity. Online learning: models can't update their weights from experience at inference time - every session starts from scratch. These two may ultimately share a single solution (E2E-TTT is a promising research direction). Hallucinations: being addressed via grounding (RAG, tool use, citations) and progress on world models. Everything else - tool access, workflow encoding, safety, parallelism - is tractable engineering moving at extraordinary speed.

Claude Code

From copilot to coworker

Open Standards

  • MCP
  • Skills

Key Capabilities

  • Long autonomous sessions
  • Computer use
  • Granular permission controls
Claude Code established the pattern now reshaping everything: give an AI agent real tools and let it do work, not just talk about work. MCP (Model Context Protocol) is adopted across the ecosystem (ChatGPT, VS Code, Goose). Skills follow the open Agent Skills standard - hot-reload, public repository, cross-tool. Parallel teams: up to 5 sub-agents on shared codebases - Anthropic built a C compiler this way. Context compaction: PreCompact hooks cut info loss by ~30%. Permission hooks expanded to 14 lifecycle events with three handler types (bash, LLM-based, full agentic verifiers). This is where the "copilot" metaphor breaks down - we're talking about delegation and orchestration, not autocomplete.

Claude Cowork

Research preview - January 2026

  • Claude Code's agent patterns, for non-developers

Coworker, not chatbot.

Cowork brings Claude Code's agent patterns to non-developers: scoped folder access, autonomous file management, parallel task queuing, external system connections via MCP and Skills. Available on Pro, Team, and Enterprise plans. The interaction model shifts from chatbot to coworker - you assign work and review results. This is where agent patterns start reaching beyond the developer audience.

Claws 🦞

Karpathy - February 2026

LLMs → LLM Agents → Claws

  • Persistent AI orchestrator wired to your tools
  • Built on MCP & Skills

OpenClaw

NanoClaw

Karpathy's layering: LLMs, then LLM Agents, now Claws - taking orchestration, scheduling, context, and persistence to the next level. A Claw is a locally-hosted AI assistant wired to your personal tools. OpenClaw (~400K lines, moving to open-source foundation) vs NanoClaw (~4K lines, "fits in my head and that of AI agents"). Karpathy rightly flags the security risk of giving AI persistent access to your email, calendar, and credentials - which motivates the "own account" model on the next slides.

  • Digital transformation on steroids

Key insight: AI agents work so well for coding because code is structured and verifiable - tests, compilers, type checkers. Non-coding knowledge work (emails, documents, planning) lacks this. But AI itself will close this gap, using its own SWE capabilities to accelerate digital transformation already underway - structuring previously unstructured work, making it API-addressable and testable. In the medium term, most knowledge work will be handled by API and agent-to-agent communication. For genuine edge cases, AI can escalate - including spawning human contractors via platforms like Upwork.

AI Workers, Not AI Assistants

  • AI workers with their own accounts
  • Assign work, review output
  • OpenAI Frontier (Feb 2026)

The plumbing is simple. The cultural shift is the bigger part.

👤 You
assign & share ↓
🤖 AI Worker
🔑 Scoped access
↑ reports & escalates
Own identity, bounded permissions
Natural evolution from the Claws security concern: instead of giving AI access to your personal accounts, give it its own accounts - own email, calendar, project management access. Share specific items as you would with a human colleague. OpenAI Frontier (Feb 2026) validates this direction - deployed at Uber, Intuit, State Farm. Accountability is straightforward: AI accounts tagged as such, linked to the human principal who controls them. The METR trajectory grounds this concretely: doubling every ~123 days means week-long autonomous tasks are plausible by late 2026.

Framing from Steve Yegge: 2024 = Chatbot, 2025 = Agent, 2026 = Orchestration. His Gas Town framework runs 20-30 parallel Claude Code instances organized into hierarchical colonies. Work expressed as "Beads" (atomic units) in "Molecules" (workflow graphs), persisted in Git. The developer role transforms from individual coder to factory operator managing agent swarms.

Discussion Points

  • How should courses shift from prompting to orchestration?
  • If AI structures all work, what roles are we training people for?
  • Does "professional competence" become management and oversight?
  • How do we teach when "Clawdbot" is 4 months old and already renamed?
These are provocations for the group, not recommendations. The pace point is real: Clawdbot launched Nov 2025 and was already rebranded to OpenClaw by Feb 2026. Six months ago, "Claws" wasn't a concept and Cowork didn't exist. How do we build curriculum that stays relevant in a landscape moving this fast?
